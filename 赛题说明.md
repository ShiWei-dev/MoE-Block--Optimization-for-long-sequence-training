# 长序列训练MoE Block显存优化
## 一、赛题描述
随着数字金融业务中复杂文本理解、多轮交互和长上下文推理需求不断提升，长序列训练已成为大模型研发中的重要方向。显存开销受模型规模、序列长度影响，同时也是决定训练吞吐的重要因素，尤其在长序列场景下，MoE 层中大量 expert 的中间激活张量以及 token 路由分发产生的临时张量会占用大量显存，容易成为训练效率优化的瓶颈。针对这一问题，招商银行在前沿技术探索与工程实践中，持续围绕训练显存优化开展相关研究，通过对中间激活、计算流程和显存生命周期等环节进行优化，在保证数值正确性的前提下显著降低显存开销，为金融场景下大模型研发与落地提供支撑。

本次赛题要求参赛者围绕给定的MoE（Mixture of Experts）Block基础实现，在单卡环境下完成"训练时显存优化"任务。参赛者无需改变模型数学定义，也不限定必须采用某一种具体优化手段；在保证数值正确性的同时，降低峰值显存占用，并尽可能控制计算耗时。

本次赛题重点关注如下方面：

对显存峰值来源和中间张量生命周期的分析能力。
在正确性、显存占用和计算速度之间进行工程权衡的能力。
构建可复现、可验证优化方案的能力。
赛题基础模块定义在baseline.py中的 MoEBlockBaseline 类，参赛者需要围绕该结构进行显存优化，配置如下：
```
{
  "hidden_size": 2048,
  "moe_intermediate_size": 768,
  "intermediate_size": 6144,
  "num_experts": 128,
  "num_experts_per_tok": 8,
  "norm_topk_prob": true,
  "torch_dtype": "bfloat16"
}
```
参赛者需基于给定的基础代码，在不改变数学定义、保持计算结果等价的前提下，实现显存优化版本。运行说明、评测方式与评分细则见附件.zip(请点击文字下载文件）。

## 四、评审规则
组织方将提供统一的 baseline 实现、输入数据、模型参数和评测入口。参赛者提交的实现将在统一单卡环境中执行，并与 baseline 进行对比。

评测时，重点比较如下指标：

1. 前向输出一致性。
2. 输入梯度一致性。
3. 参数梯度一致性。
4. 峰值显存占用。
5. 单步训练耗时。

组织方将在多组测试样例上执行前向和反向计算。正确性与性能均纳入总成绩计算。组织方提供正确性自测脚本correctness_check.py用于快速检查实现与baseline在前向输出、输入梯度、参数梯度是否一致；提供性能自测脚本benchmark.py用于本地测试实现的峰值显存占用和单步训练耗时，帮助参赛者在竞赛过程中及时评估峰值显存占用、平均单步耗时及最小/最大单步耗时等优化效果。

本次赛题采用"正确性门槛 + 显存优化评分 + 速度评分"的方式进行评测。特别注意：只有在正确性完全通过（即前向输出、输入梯度、参数梯度一致性全部达标）的前提下，才会进行显存优化和速度的评测与打分。若正确性未通过，显存与速度得分记为 0 分。

（一）正确性门槛
正确性门槛由以下三项组成：

1. 前向输出一致性。
2. 输入梯度一致性。
3. 参数梯度一致性。
每一项均在多组测试样例上独立统计通过情况。一致性判定的浮点误差容忍范围如下：
- 前向：`rtol=2e-2, atol=1e-3`
- 梯度：`cosine_sim >= 0.995` 且 `relative_l2 <= 1e-2`
测试输入规模、评测轮数及具体权重由组织方统一设定。

（二）显存优化评分
组织方将使用统一评测脚本测量峰值显存占用。评测方式包括但不限于：

1. 在固定输入规模（8k和128k长度）下，分别执行baseline实现与参赛者实现。
2. 通过统一的PyTorch显存统计方式获取峰值显存。
3. 根据相对于baseline实现的显存下降比例计算显存优化得分。
显存优化越明显，得分越高。

（三）速度评分
组织方将使用统一评测脚本测量单步训练平均耗时。评测方式包括但不限于：

1. 固定 warmup 轮数。
2. 固定正式评测轮数
3. 统计平均step time。
4. 输入规模为8K和128K长度。
在显存优化效果相近的前提下，耗时越低，得分越高。

（四）总成绩计算方式
在测试样例正确性通过的前提下，总成绩由显存优化评分和速度评分累计加权构成；若某组样例正确性测试未通过，则该样例显存优化评分和速度评分计为0分。

1. 显存优化评分：反映相对于 baseline 实现的峰值显存下降幅度。
2. 速度评分：反映相对于 baseline 实现的单步训练耗时表现。
3. 显存优化评分与速度评分的权重比例为6:4。

若多位参赛者总成绩相同，则优先比较显存优化得分。若显存优化得分仍相同，则比较速度得分。
